## サービス名
# 「Miniita(ミニータ)」

サービスURL: [Miniita](https://www.miniita.com/)
[![Image from Gyazo](https://i.gyazo.com/f9acd6ce27eb2065aac3cfb78aee5c24.png)](https://gyazo.com/f9acd6ce27eb2065aac3cfb78aee5c24)

## ■サービス概要
「Miniita（ミニータ）」は技術記事を書いたことがない / 投稿したいけどハードルが高く投稿できないという方向けに、技術記事を投稿しやすい環境を提供するアプリです。QiitaやZennで将来的に技術記事が書けるようになるために、Miniitaでは「①書くことへの抵抗感を下げる ②見られることへの抵抗感を下げる ③継続して投稿を続けられる」機会をユーザーに提供します。

## ■ このサービスへの思い・作りたい理由
私自身、技術記事を投稿したくQiitaで執筆していましたが、投稿へのハードルの高さから断念してしまった経験がありこのサービスを発案しました。また初学者の方の多くが、自分のNotionに学習内容をまとめており外部へ公開している方は少ないと感じました。Miniitaのように外部にアウトプットしやすい技術記事アプリを作ることで、初学者でも気軽に投稿できると考え開発を決意しました。

## ■ ユーザー層について
- 技術記事を投稿するのにハードルを感じている方
- 自分の記事内容を人に見られることに抵抗を感じている方
- 継続して記事を投稿できていない方

## ■サービスの利用イメージ
#### MiniitaBotによる、記事作成段階でのサポート機能
**投稿するハードルを下げる**ポイントです。記事を投稿し終えるまでのハードルの高さの1つとして、記事作成段階で不安になってしまうことが挙げられます。
MiniitaBotでは、この不安を解消するため予め質問選択肢に下記3つを織り込んでいます。
- **「テンプレート作成依頼」**： 技術記事を書き慣れていない場合、どのような形式に沿って書けばいいか迷ってしまう可能性が高いと考え、この質問項目を設けました。
自分の投稿したい内容に合わせ、MiniitaBotが3-5項目程度の項目を提示してくれます。その内容を参照しながら記事を書くことができます。
- **「記事の内容の確認依頼」**： 記事内容に不安があると、投稿するのに躊躇してしまいます。
不安がある箇所を入力欄へ貼り付け、MiniitaBotへ内容の確認依頼をすることが可能です。
- **「誤字脱字の確認依頼」**： 記事内で誤字脱字がないか、MiniitaBotへ内容の確認依頼をすることが可能です。

#### いいねボタン
**見られるハードルを下げる**ポイントです。
類似サービスではいいねボタンは1種類ですが、Miniitaでは５種類用意し、記事のどの点が他のユーザーの参考になったのかを一目見てわかる仕様にしています。
現時点では、以下のボタンが設置されています。
- 参考になった
- 学びが深まった
- 解説が丁寧
- 独自の視点
- 詳細をもっと知りたい
ユーザーは他のユーザーからこの通知を受け取り、記事内容を客観的に見ることができます。
記事内容でどの点がユーザーにいいねと思ってもらえたのかを知ることで、次回以降の記事投稿の意欲上昇にも貢献します。

#### マイバッジ機能
**継続して投稿を続ける**ポイントの1つ目です。
1,3,5,10,20,30記事を書き終えた段階で、バッジがマイページ上で付与されます。
バッジ内容はXでシェアすることも可能です。バッジが付与されることで、継続的に記事を投稿し続けるモチベーションに繋がると考え、ユーザーバッジ機能を作成しました。

#### 継続サポート機能
**継続して投稿を続ける**ポイントの2つ目です。
マイページ上に「継続サポート」という項目を設置しました。
この画面では、**総いいね獲得数/総文字数（タイトル除く）/MiniitaBotからの総評(5記事以上投稿した場合のみ利用可能)**という項目を設置しています。
- **総いいね獲得数**： 自分の記事に幾ついいねボタンが押されたのか、その総数を確認できます。各いいねボタンでの内訳も記載しているため、これまでの記事がどんな点で他のユーザーの役に立ったのか確認することが可能です。
- **総文字数（タイトル除く）**： これまで投稿した記事の総文字数を確認できます。日々記事をどの程度投稿したのかが可視化されることで、今後のモチベーションアップに貢献します。
- **MiniitaBotからの総評**： 5記事以上投稿したユーザー限定で使用できる機能です。これまで書いたタイトルの総評をしてもらえます。（ChatGPTのMaxtoken、金銭的な制約上、本文総評は今回は機能へ含めておりません）

## ■ ユーザーの獲得について
#### X（旧Twitter）から
Xシェアできる内容を下記に設定し、ユーザーの目に止まる確率を増やしました。
- 投稿内容のシェア
- ユーザーが獲得したバッジをシェア
  
#### Qiitaへの開発記事投稿から
以下の内容をQiitaやZennに投稿することで、ユーザーを確保できると考えています。
- 技術記事を書くのを断念してしまった私自身が、Miniitaで技術記事を書く練習をした結果、Qiitaにも堂々と投稿できるようになった話
- Miniitaの開発秘話

## ■ サービスの差別化ポイント・推しポイント
【比較したサービス①】： 技術記事に特化した投稿サイト  
[Qiita](https://qiita.com/) / [Zenn](https://zenn.dev/) / [DEV community](http://dev.to/)  
【差別化ポイント】
- 記事投稿時にMIniitaBotが用意されている点。
-> Miniitaは「技術記事を書くことに抵抗がある人」向けのサービスです。その場で不明点を確認できようMiniitaBotを用意しています。

【比較したサービス①】： 記事全般を投稿できるサイト  
[はてなブログ](https://blog.hatena.ne.jp/s0917w/s0917w.hatenablog.com/edit?utm_source=service_globalnav_pc&utm_medium=referral&utm_campaign=new_entry&_gl=1*1hph8f6*_gcl_au*MTQzMTEwOTEyNy4xNzE2NTM5NTY5*_ga*MTQzMDM3MDE1Ny4xNzE2NTM5NTY5*_ga_3WN9D6N5N5*MTcxNjUzOTU2OS4xLjEuMTcxNjUzOTYwMi4yNy4wLjA.) / [Medium](https://medium.com/) / [note](https://note.com/)

【差別化ポイント】
- いいね機能が複数用意されている点。
-> 記事のどの部分に共感していいねボタンが押されたか、いいねボタンの内容によってわかります。
-> このアプリは技術記事を見られるハードルが低いため、このアプリを使用することでユーザーは「自分の記事のどの部分を評価してもらったのか」を理解し、次に技術記事を投稿する意欲を掻き立てられます。
